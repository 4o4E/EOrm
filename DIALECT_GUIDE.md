# 添加数据库方言集成指南

本文档说明如何为 EOrm 添加新的数据库方言支持。

## 项目结构

每个数据库方言是一个独立的 Gradle 模块：

```
EOrm/
├── eorm-core/                    # 核心库（SqlDialect 接口、BaseDialect 基类）
├── eorm-dialect-mysql/           # MySQL 方言
├── eorm-dialect-postgres/        # PostgreSQL 方言
├── eorm-dialect-sqlite/          # SQLite 方言
├── eorm-dialect-h2/              # H2 方言
└── eorm-dialect-xxx/             # ← 你的新方言
```

## 快速开始

以添加 SQL Server 方言为例。

### 1. 创建模块

在 `settings.gradle.kts` 中注册：

```kotlin
include("eorm-dialect-sqlserver")
```

创建 `eorm-dialect-sqlserver/build.gradle.kts`：

```kotlin
dependencies {
    implementation(project(":eorm-core"))
}
```

### 2. 实现方言类

创建 `eorm-dialect-sqlserver/src/main/kotlin/top/e404/eorm/dialect/SqlServerDialect.kt`：

```kotlin
package top.e404.eorm.dialect

import top.e404.eorm.generator.IdStrategy

class SqlServerDialect : BaseDialect() {

    // ==================== 必须覆写 ====================

    override fun wrapName(name: String): String = "[$name]"

    override fun getSqlType(type: Class<*>, length: Int): String {
        return when (type) {
            Boolean::class.java, java.lang.Boolean::class.java -> "BIT"
            String::class.java -> "NVARCHAR($length)"
            Int::class.java, Integer::class.java -> "INT"
            Long::class.java, java.lang.Long::class.java -> "BIGINT"
            Double::class.java, java.lang.Double::class.java -> "FLOAT"
            java.util.Date::class.java -> "DATETIME2"
            java.time.LocalDateTime::class.java -> "DATETIME2"
            java.time.LocalDate::class.java -> "DATE"
            java.time.LocalTime::class.java -> "TIME"
            else -> "NVARCHAR(255)"
        }
    }

    override fun getPrimaryKeyDefinition(strategy: IdStrategy): String {
        return if (strategy == IdStrategy.AUTO) "PRIMARY KEY IDENTITY(1,1)" else "PRIMARY KEY"
    }

    // ==================== 按需覆写 ====================

    override fun supportsIfNotExists(): Boolean = false

    override fun buildPaginationSql(sql: String, offset: Long, limit: Long): String {
        // SQL Server 2012+ 支持 OFFSET FETCH
        return "$sql OFFSET $offset ROWS FETCH NEXT $limit ROWS ONLY"
    }

    override fun buildLimitSql(sql: String, limit: Int): String {
        // SQL Server 使用 TOP
        return sql.replaceFirst("SELECT ", "SELECT TOP $limit ")
    }
}
```

### 3. 使用

```kotlin
dependencies {
    implementation("top.e404.eorm:eorm-core:1.0.0-SNAPSHOT")
    implementation("top.e404.eorm:eorm-dialect-sqlserver:1.0.0-SNAPSHOT")
}
```

```kotlin
val db = EOrm(dataSource, SqlServerDialect())
```

## SqlDialect 接口方法参考

`BaseDialect` 为所有方法提供了默认实现，新方言只需覆写与目标数据库不兼容的方法。

### 必须覆写

这三个方法没有通用默认值，每个数据库都不同：

| 方法 | 说明 | 示例 |
|---|---|---|
| `wrapName(name)` | 标识符包装 | MySQL: `` `name` ``，PG: `"name"`，SQL Server: `[name]` |
| `getSqlType(type, length)` | Java 类型 → SQL 类型 | `String` → MySQL: `VARCHAR(n)`，SQLite: `TEXT` |
| `getPrimaryKeyDefinition(strategy)` | 自增主键定义 | MySQL: `AUTO_INCREMENT`，PG: `GENERATED BY DEFAULT AS IDENTITY` |

### 按需覆写

以下方法在 `BaseDialect` 中有合理的默认实现，仅在目标数据库语法不兼容时覆写：

| 方法 | 默认实现 | 何时需要覆写 |
|---|---|---|
| `valueToSql(value)` | 标准 SQL 字面量（`'str'`、`NULL`、数字原样） | 数据库有特殊字面量语法时（如 PG 的 `TRUE`/`FALSE`） |
| `buildPaginationSql(sql, offset, limit)` | `LIMIT $limit OFFSET $offset` | SQL Server: `OFFSET FETCH`，Oracle: `FETCH FIRST` |
| `buildLimitSql(sql, limit)` | `$sql LIMIT $limit` | SQL Server: `SELECT TOP n ...` |
| `supportsIfNotExists()` | `true` | SQL Server/Oracle 不支持，返回 `false` |
| `buildInsertSql(tableName, columnNames)` | `INSERT INTO t (c1, c2) VALUES (?, ?)` | PG 追加 `RETURNING id`，MySQL 追加 `ON DUPLICATE KEY UPDATE` |
| `prepareInsertStatement(conn, sql, idColumnName)` | `conn.prepareStatement(sql, RETURN_GENERATED_KEYS)` | Oracle: `conn.prepareStatement(sql, arrayOf("ID"))` |
| `extractGeneratedKeys(stmt, entities, idField, convertType)` | 通过 `stmt.generatedKeys` 逐行读取 | PG 使用 RETURNING 时从普通结果集读取 |
| `getInsertBatchSize()` | `100` | 根据数据库和驱动调优（MySQL 可设更大，SQLite 可设更小） |

## 各数据库覆写参考

### MySQL

最简单的方言，只需覆写三个必须方法。`BaseDialect` 的默认行为与 MySQL 完全兼容。

```kotlin
class MySqlDialect : BaseDialect() {
    override fun wrapName(name: String) = "`$name`"
    override fun getSqlType(type: Class<*>, length: Int) = ...
    override fun getPrimaryKeyDefinition(strategy: IdStrategy) =
        if (strategy == IdStrategy.AUTO) "PRIMARY KEY AUTO_INCREMENT" else "PRIMARY KEY"
}
```

### PostgreSQL

需要额外覆写主键定义。分页语法与默认一致，无需覆写。

```kotlin
class PostgreSqlDialect : BaseDialect() {
    override fun wrapName(name: String) = "\"$name\""
    override fun getSqlType(type: Class<*>, length: Int) = ...
    override fun getPrimaryKeyDefinition(strategy: IdStrategy) =
        if (strategy == IdStrategy.AUTO) "PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY" else "PRIMARY KEY"

    // 可选：覆写 buildInsertSql 追加 RETURNING
    // override fun buildInsertSql(tableName: String, columnNames: List<String>): String {
    //     return super.buildInsertSql(tableName, columnNames) + " RETURNING id"
    // }
}
```

### SQLite

类型系统简单（INTEGER/TEXT/REAL），自增语法为 `AUTOINCREMENT`。

```kotlin
class SqliteDialect : BaseDialect() {
    override fun wrapName(name: String) = "\"$name\""
    override fun getSqlType(type: Class<*>, length: Int) = ... // 大部分映射为 TEXT 或 INTEGER
    override fun getPrimaryKeyDefinition(strategy: IdStrategy) =
        if (strategy == IdStrategy.AUTO) "PRIMARY KEY AUTOINCREMENT" else "PRIMARY KEY"
}
```

### SQL Server（示例）

差异最大，需要覆写较多方法：

```kotlin
class SqlServerDialect : BaseDialect() {
    override fun wrapName(name: String) = "[$name]"
    override fun getSqlType(type: Class<*>, length: Int) = ...
    override fun getPrimaryKeyDefinition(strategy: IdStrategy) =
        if (strategy == IdStrategy.AUTO) "PRIMARY KEY IDENTITY(1,1)" else "PRIMARY KEY"

    override fun supportsIfNotExists() = false
    override fun buildPaginationSql(sql: String, offset: Long, limit: Long) =
        "$sql OFFSET $offset ROWS FETCH NEXT $limit ROWS ONLY"
    override fun buildLimitSql(sql: String, limit: Int) =
        sql.replaceFirst("SELECT ", "SELECT TOP $limit ")
}
```

### Oracle（示例）

```kotlin
class OracleDialect : BaseDialect() {
    override fun wrapName(name: String) = "\"$name\""
    override fun getSqlType(type: Class<*>, length: Int) = ... // VARCHAR2, NUMBER, TIMESTAMP 等
    override fun getPrimaryKeyDefinition(strategy: IdStrategy) =
        if (strategy == IdStrategy.AUTO) "PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY" else "PRIMARY KEY"

    override fun supportsIfNotExists() = false
    override fun buildPaginationSql(sql: String, offset: Long, limit: Long) =
        "$sql OFFSET $offset ROWS FETCH NEXT $limit ROWS ONLY"
    override fun buildLimitSql(sql: String, limit: Int) =
        "$sql FETCH FIRST $limit ROWS ONLY"

    override fun prepareInsertStatement(conn: Connection, sql: String, idColumnName: String?): PreparedStatement {
        return if (idColumnName != null) {
            conn.prepareStatement(sql, arrayOf(idColumnName))
        } else {
            conn.prepareStatement(sql)
        }
    }
}
```

## 测试建议

为新方言编写集成测试时，推荐使用 Testcontainers 或内存数据库：

```kotlin
class SqlServerDialectTest {
    private lateinit var db: EOrm

    @BeforeEach
    fun setup() {
        val dataSource = ... // Testcontainers 或真实数据库连接
        db = EOrm(dataSource, SqlServerDialect())
        db.createTable<TestEntity>()
    }

    @Test
    fun `basic crud`() {
        val entity = TestEntity(name = "test")
        db.insert(entity)
        val fetched = db.from<TestEntity>("t").where { eq(TestEntity::id, entity.id) }.firstOrNull()
        assertEquals("test", fetched?.name)
    }
}
```

核心模块的单元测试使用 H2 内存数据库（MySQL 兼容模式），覆盖了所有 CRUD、DSL 查询、事务、子查询等功能。新方言只需验证 SQL 语法差异部分。
